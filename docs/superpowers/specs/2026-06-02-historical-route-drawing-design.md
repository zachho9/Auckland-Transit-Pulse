# Historical Data Metrics & Route Drawing — Design Spec

**Date:** 2026-06-02
**Status:** Approved

---

## What We're Building

Two post-MVP features for Auckland Transit Pulse:

1. **Historical data metrics** — a 7-day trends panel in the sidebar showing on-time % per mode, network average delay, and chronically worst routes
2. **Route drawing** — when a user selects a route (via vehicle marker or league table), draw its path on the map, highlight its active vehicles, and hide all others

---

## Architecture

Two additions to the existing CDK stack:

**S3 bucket** — stores pre-processed per-route shape JSON files generated by a new `generateShapes.ts` script. A new `GET /shapes/{routeId}` API endpoint reads from S3 and returns the shape on demand.

**DynamoDB daily-stats items** — the existing table gains a second item type written by the poller once per poll run, aggregating today's metrics. A new `GET /history` endpoint returns the last 7 daily items. The existing DynamoDB `removalPolicy` changes from `DESTROY` to `RETAIN`.

```
Amplify (React SPA)
    │  GET /snapshot     (every 30s)
    │  GET /history      (on page load)
    │  GET /shapes/{id}  (on route select)
    ▼
API Gateway → Lambda (API) ──reads──▶ DynamoDB (snapshot + daily-stats)
                           ──reads──▶ S3 (shape files)
                                ▲
EventBridge (every 1 min)       │
    │                         writes
    ▼
Lambda (Poller) ──▶ AT Realtime API
```

**Estimated cost addition:** under $0.10/month. S3 stores ~10MB of shape files; DynamoDB gains ~43,000 writes/month for daily-stats updates. Both fall within or very near AWS free tier.

## Data Model

### DynamoDB — daily-stats item

```ts
{
  pk: "daily-stats",
  sk: "2026-06-02",         // ISO date string — used for range queries
  ttl: 1749081600,          // Unix timestamp 8 days from sk — auto-deleted by DynamoDB TTL
  sampleCount: number,      // number of poll runs that have contributed to today's averages
  onTimePercent: {
    bus:   number,          // rolling daily average % on time
    train: number,
    ferry: number,
  },
  avgDelayMinutes: number,  // network-wide average across all modes
  worstOffenders: [         // routes that appeared in top 10 most frequently today
    { routeId: string, name: string, count: number }
    // top 5, sorted by count descending
  ]
}
```

The poller updates today's item on every run using an incremental average formula: `new_avg = (old_avg * sampleCount + current_value) / (sampleCount + 1)`. `sampleCount` increments each run. `worstOffenders` counts are incremented for any route present in the current snapshot's top 10. TTL is set to 8 days (not 7) to ensure items are still present when queried on the 7th day, given DynamoDB TTL deletion can lag up to 48 hours. The `GET /history` endpoint queries `pk = "daily-stats"` for the last 7 `sk` date values and returns them sorted oldest → newest.

### S3 — shape file `shapes/{routeId}.json`

```ts
{
  routeId: "101-202",
  directions: [
    {
      directionId: 0,
      points: [[-36.849, 174.544], [-36.848, 174.545], ...]  // [lat, lng] pairs
    },
    {
      directionId: 1,
      points: [...]   // omitted if only one direction exists in GTFS data
    }
  ]
}
```

Points are simplified using the Douglas-Peucker algorithm to reduce file size. Raw shapes average ~2,600 points per route; simplification targets ~300–500 points with no visible quality loss at the map zoom levels used.

The `generateShapes.ts` script joins `trips.txt` (route_id → shape_id, direction_id) and `shapes.txt` (shape_id → ordered lat/lon points), picks one representative trip per direction, simplifies the points, and writes one JSON file per route to S3.

### `VehicleSnapshot` type addition

```ts
interface VehicleSnapshot {
  id: string;
  lat: number;
  lng: number;
  delaySeverity: DelaySeverity;
  mode: TransitMode;
  routeShortName: string;
  routeId: string;           // added — needed to match vehicle to selected route and shape file
}
```

## Frontend

### Right Panel Redesign

The sidebar becomes a structured scrollable container with collapsible sections. Each section has a header with a toggle chevron. Collapsed sections show only their header. The scorecard is always visible at the top.

```
┌─────────────────────────────┐
│  NETWORK SCORECARD          │  ← always expanded, not collapsible
│  Bus 94%  Train 81%  Ferry 97%
│  312 active                 │
├─────────────────────────────┤
│  WORST ROUTES          [^]  │  ← collapsible, default expanded
│  1. 274  +8 min             │
│  2. 380  +6 min  ...        │
├─────────────────────────────┤
│  7-DAY TRENDS          [^]  │  ← collapsible, default expanded, new
│  [on-time % line chart]     │
│  [avg delay line chart]     │
│  [worst offenders list]     │
├─────────────────────────────┤
│  SERVICE ALERTS        [^]  │  ← collapsible, default expanded
│  ⚠ Northern busway...       │
└─────────────────────────────┘
```

### Historical Charts (7-day Trends Panel)

Three sub-sections:

- **On-time % by mode** — three lines (bus/train/ferry) in the existing green/amber/red colour scale over 7 daily data points
- **Network avg delay** — single line in minutes over 7 daily data points
- **Worst offenders** — ranked list: route name + number of days it appeared in top 10 over the last 7 days (not a chart)

Chart library: **Recharts** — lightweight, React-native, minimal configuration.

If fewer than 7 days of data exist (e.g. first week after deploy), the chart renders with however many points are available. No error shown.

### Route Selection & Map

**How a route gets selected:**
- Clicking a vehicle marker → selects that vehicle's `routeId`
- Clicking a worst-routes league table item → selects its `routeId`

**On selection:**
1. Fetch `GET /shapes/{routeId}` from API
2. Draw a `Polyline` per direction in react-leaflet, coloured by the route's current `avgDelayMinutes` mapped to green/amber/red
3. Hide all vehicles not on the selected route
4. Pan and zoom the map to fit the route's bounds

**On deselection:**
- Clicking anywhere on the map (existing `MapClickHandler`) clears the selected route, removes polylines, restores all vehicles

**State:** `selectedRouteId: string | null` lives in a new `RouteSelectionContext` so both `MapPanel` (renders polyline, filters vehicles) and `LeagueTablePanel` (triggers selection) can read and write it without prop drilling.

## Error Handling

| Scenario | Behaviour |
|---|---|
| `/history` fetch fails | Hide 7-day trends panel, show "Trends unavailable" in its place. Other panels unaffected. |
| `/shapes/{routeId}` fetch fails | Skip polyline drawing, still filter vehicles to selected route. No error shown to user. |
| Shape file not found (404) | Same as fetch failure — silently skip polyline. |
| Daily-stats DynamoDB write fails | Log error, continue — snapshot write is unaffected. One poll cycle missing from daily average is negligible. |
| Fewer than 7 days of history | Chart renders with available data points, no error shown. |

## Testing

- **Unit tests** — extend existing poller tests to cover daily-stats aggregation: rolling average calculation, worst-offender frequency counting, TTL timestamp generation
- **`generateShapes.ts` script** — manual verification: run script, spot-check a known route's output JSON against raw `shapes.txt` entries
- **Manual smoke test** after deploy — verify `/history` returns 7 items, `/shapes/{routeId}` returns valid GeoJSON, route polyline renders on map, all panels collapse/expand correctly

## Out of Scope

- Sub-daily historical granularity (hourly trends)
- Historical data export
- Route shape animation (moving polyline)
- Mobile layout adjustments for redesigned sidebar
